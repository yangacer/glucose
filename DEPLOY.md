# JavaScript Build Workflow

## Overview
This project uses a dev/prod split for JavaScript files to support both easy debugging and optimized production delivery.

## File Structure

- **`static/index.html.dev`** - Development HTML with individual `<script>` tags
  - Single source of truth for script dependency order
  - Used during development for debugging
  - Never minified, easy to read browser console errors

- **`static/index.html`** - Production HTML with minified bundle
  - Auto-generated by `build-js.py`
  - References `js/release/app.min.js?v=x.y.z`
  - Used in production for performance

- **`static/js/release/app.min.js`** - Minified JavaScript bundle
  - Auto-generated by `build-js.py`
  - Combines all JS files (except chart.js and *.min.js)
  - Minified with terser

## Development Workflow

### Option 1: Development Mode (Recommended for active development)
1. Use `static/index.html.dev` as your HTML file
2. Edit individual JS files in `static/js/`
3. Refresh browser to see changes immediately
4. No build step needed during development

**To use dev mode:**
```bash
# In server.py or your web server, serve index.html.dev instead of index.html
# OR create a symlink:
ln -sf index.html.dev static/index.html
```

### Option 2: Production Mode Testing
1. Edit JS files in `static/js/`
2. Run `./build-js.py` to rebuild bundle
3. Test with minified version

**When to build:**
- Before deploying to production
- Before committing (optional)
- When testing production performance
- When updating version number

## Production Workflow

Before deploying:
```bash
# Build with current version
./build-js.py

# Or bump version
./build-js.py 0.6.0
```

This generates:
- `static/js/release/app.min.js` - Minified bundle
- `static/index.html` - Production HTML pointing to bundle

## Updating Script Order

If you need to change the load order (e.g., add new dependencies):

1. Edit `static/index.html.dev` - add/reorder `<script>` tags
2. Run `./build-js.py` - automatically picks up new order
3. Test both dev and prod modes

## Summary

**Development:** Use `index.html.dev` → No build needed
**Production:** Run `./build-js.py` → Deploy with `index.html`

The build script is **NOT** required during active development.
Only run it when preparing for production deployment.
